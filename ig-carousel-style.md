## 二、P1 封面公式

### 結構

```
┌─────────────────────────────────────┐
│                                     │
│      ［視覺主圖區，約占 75%］        │
│                                     │
│  ─ 兩種選項 ─                       │
│  A. 個人實拍 + 浮動 Logo/icon        │
│  B. AI 生成概念場景（棋盤/宇宙等）   │
│                                     │
├─────────────────────────────────────┤
│  ▒▒▒▒▒  白色斜切色塊  ▒▒▒▒▒          │
│  ▒  橘黃色主標（最多 2 行）  ▒        │
│  ▒  每行 8-15 字              ▒        │
├─────────────────────────────────────┤
│       IG @on.chain.with.jerry       │
└─────────────────────────────────────┘
```

### 兩種視覺風格

| 風格 | 用法 | 範例 |
|------|------|------|
| **A. 個人實拍 + 浮動元素** | 比較柔軟、有作者人格 | 危險邊緣的美元（Jerry 在瀑布前 + USDT/PYUSD/USDC/DAI 浮動）｜ AP2（Jerry 在幾何穹頂下） |
| **B. AI 生成概念場景** | 比較抽象、議題重型 | Coinbase 萬物交易所（宇宙球體 + C logo）｜ 穩定幣核心戰場（西洋棋盤 + 加密貨幣符號）|

### 標題公式（依題目類型選用）

| 題目類型 | 公式 | 範例 |
|----------|------|------|
| **爭議題** | 「主標：副標問句？」 | 「穩定幣核心戰場解析：是金融革命還是主權威脅？」 |
| **解釋題** | 「從 X 到 Y：定論短句」 | 「從交易所到金融基礎設施：Coinbase 的萬物交易所計畫」 |
| **預測題** | 「未來 X 年的 Y？...」 | 「未來十年的 Alpha？Google 新支付協議 AP2」 |
| **抒情題** | 「主標，副標問句」 | 「危險邊緣的美元，穩定幣能成為救命稻草嗎？」 |

> 共通點：**封面標題一定要有資訊量或張力**，不能是空泛口號。

## 三、內頁公式（5 種型態）

### 型態 A：標題 + 多卡片網格

**用於**：列舉多個案例 / 多國比較 / 多家公司動態

**結構**：
- 頂端：黑色粗體標題（無框）
- ▶ 橘色三角箭頭 + 子標題（橘色加粗）
- 2x2 卡片網格，每卡含 icon/國旗 + 一句話論述（30-50 字）
- 下方可加：數據視覺化（折線圖 + 大字數據 highlight）

**範例**：P2 美元霸權正面臨多重挑戰（4 個國家應對措施 + 美債圖表）

---

### 型態 B：對照表（**最強招牌結構**）

**用於**：概念比較 / 多方案對比

**結構**：
- 頂端：黑色粗體標題
- 副標題：一句話定調（淺灰色）
- 多欄對照表（背景淺灰、欄頭加色）
- 每格短文字 + 重點顏色標記
- 表格下方一句結論

**範例**：穩定幣 / CBDC / 代幣化存款比較（六行特性 × 三欄對比）

> 這個型態是鏈傑風格的最強記憶點——複雜概念變得一眼能讀懂。

---

### 型態 C：精準論點拆解（A.A.A. 結構）

**用於**：技術／概念深度拆解

**結構**：
- 頂端標題（標題內有引號精準短句）
- 一段引語（白色卡片內，淺灰）：用問題或場景白話化
- 三點論述：**用大寫字母（A/A/A）+ 中文翻譯**（如 Authorization 授權 / Authenticity 真實性 / Accountability 問責）
- 解決方案區：**橘色圓底數字 1/2/3** + 短描述

**範例**：P5 解鎖信任「是 AI 幹的，但你同意的」（AAA 信任問題 + 三種授權信任）

---

### 型態 D：對立敘事（兩立場並列）

**用於**：辯證題 / 雙面影響分析

**結構**：
- 頂端問句標題（「...是威脅還是增長引擎？」）
- 兩個並列**圓角矩形按鈕**（一邊綠青、一邊紅橘）標立場
- 各立場下一段論述（每段 30-60 字）
- 中間有一條**垂直曲線**做視覺切割
- 底部**白色卡片** + 引號 + 粗體**結論金句**（這是鏈傑的精煉招牌詞發力點）

**範例**：P3 對傳統銀行是威脅還是增長引擎？（大量活水 vs 存款替代 + 「適者生存」+ BaaS 2.0 概念）

---

### 型態 E：總結 / 總覽

**用於**：carousel 倒數第二頁的總攬

**結構**：
- 頂端標題（如「穩定幣戰場總覽」）
- 大型對照表（多列多欄，含拉攏力／未來預判）
- 表格下方：**橘色圓形標籤** + 趨勢名稱（2-3 個大趨勢）
- 標籤下方各一行行動建議

**範例**：P7 穩定幣戰場總覽（六大戰場拉攏勢力對照 + 三大趨勢）

## 四、末頁收尾公式（**多選項，不必死守鏈傑觀察**）

### 末頁可選的三種收尾型態

| 型態 | 用法 | 範例 |
|------|------|------|
| **A. 鏈傑觀察 / 鏈傑洞察 / 鏈傑提醒** | 文字論述為主，分析事件意義 | 十年以太坊 P5、Circle P8 |
| **B. 比較表 / 戰場總覽** | 對照多方勢力、羅列各情境 | 穩定幣核心戰場 P7 戰場總覽 |
| **C. 趨勢標籤 + 行動建議** | 橘色圓形標籤列大趨勢 + 一行行動 | 戰場總覽下方「擁抱合規／BaaS 轉型／深耕全球南方」 |

> 設計原則：**依題目自然度選用**，不該硬塞鏈傑觀察。比較題、競爭題、總覽題用 B/C 型態通常更有力。

---

### 鏈傑觀察 / 鏈傑洞察 / 鏈傑提醒 公式（型態 A 詳解）

### 結構

```
┌─────────────────────────────────────┐
│           鏈傑觀察 / 鏈傑洞察         │
│           ─────────────             │
│                                     │
│   ▶ 副標 1（橘色加粗）               │
│   一段論述（完整段落，非條列點）      │
│                                     │
│   ▶ 副標 2                          │
│   一段論述                           │
│                                     │
│   （可選：引號內精煉金句獨立成卡）   │
│                                     │
│       IG @on.chain.with.jerry       │
└─────────────────────────────────────┘
```

### 文案特徵

- **這是 IG carousel 才有的署名頁**（LinkedIn 沒有對應頁——LinkedIn 的判斷融在末段）
- 文字密度比內頁高（約 200-300 字）
- 用 ▶ 橘色三角分 2-3 個觀察點
- 段落是**完整論述**，不是 bullet points
- **可插入引號金句卡**強化記憶點（例：「支付的未來不只在於發行貨幣，更在於掌握底層的基礎設施。」）
- 結尾是**判斷句／預測句**，不另寫「總結」transition

### 命名差異

- **鏈傑觀察**：用於分析現況、解讀事件
- **鏈傑洞察**：用於更深層的趨勢判斷（例：Circle 從發行商到生態主導）
- **鏈傑提醒**：用於有風險／需注意的議題

## 五、限時動態（Story）公式

### 結構

- **9:16 直式**（與 carousel 不同）
- 頂端：橘色粗體大標（最大字級）
- **「資訊重點」**：黑色加粗子標 + 一段乾貨論述
- **「白話解讀」**：黑色加粗子標 + 一段接地氣翻譯
- 右下角：**日期戳記**（YYYY-MM-DD）
- 底部：IG handle watermark

### 文案特徵（**重要發現**）

限時動態的「資訊重點 / 白話解讀」**雙段式**是固定模板：
- 「資訊重點」段：給乾貨數據 + 帶引號的精煉概念（如「實體經濟」「支付痛點」）
- 「白話解讀」段：用比喻或場景拉近讀者（如「目前還處在一個尷尬期，雖然大家在鏈上轉來轉去」）

> 這個模板可獨立成 `lianjie-story` skill 的核心結構。

## 六、IG Carousel vs LinkedIn 文案差異對照

| 維度 | LinkedIn 中長文 | IG Carousel |
|------|-----------------|-------------|
| **總字數** | 800-1100 字（連續敘事） | 全頁加總 600-1000 字（散在各頁）|
| **transition 句** | 用「真正的痛點在於...」承轉 | 取消 transition，靠視覺結構分隔 |
| **▌符號** | 用，分主題段 | 不用（▌只在 LinkedIn）|
| **▶ 橘色三角** | 不用 | 用，副標前 |
| **段落感** | 段段相連、敘事流暢 | 每張圖獨立成立、條列化 |
| **概念精煉招牌詞** | 內嵌段落中（甜蜜點、原子化）| 獨立成標籤、引號金句卡（BaaS 2.0、適者生存）|
| **括號吐槽** | 多用（很現實）（已被 Stripe 收購）| 也用，但更精簡 |
| **第一人稱** | 直接寫「我認為」「我蠻看好」| 末頁鏈傑觀察才出現 |
| **對讀者說「你」** | 用 | 用，更頻繁（「你的加密貨幣由 Coinbase 託管」）|
| **末頁署名** | **無**（判斷融在末段最後一句）| **有**（顯式「鏈傑觀察 / 鏈傑洞察 / 鏈傑提醒」頁）|
| **結構表達** | 段落 + ▌主題 | 表格 / 卡片 / 三點 / 並列 |

## 七、共通的「鏈傑風格」（IG + LinkedIn 都有）

不論平台都會出現的核心 DNA：

1. **概念精煉招牌詞**：甜蜜點、原子化、白牌、超 App、萬物交易所、日不落、BaaS 2.0、適者生存、實用性鴻溝
2. **三角分析框架**：法律 / 商業 / 技術（多面向同時）
3. **代入式設問**：站在...、對於...、想像一下...
4. **對比強化**：傳統 vs 新、自建 vs 服務化、威脅 vs 增長引擎
5. **數據錨點**：每論點都有具體數字 / 公司名 / 案例
6. **括號吐槽**：（很現實）（已被 Stripe 收購）（例如美國公債）
7. **引號標記重點概念**：「鏈上原生」「BaaS 2.0」「支付痛點」「實用性鴻溝」

## 八、Storyboard / Visual skill 的關鍵設計指引

### Storyboard skill 必須做的事

接到 Jerry 確認過的定稿後：

1. **判斷封面公式類型**（A 個人實拍 / B AI 場景）依題目氛圍決定
2. **依論點數量決定總頁數**（3-10 頁彈性）
3. **替每頁分配內頁型態**（A 卡片 / B 對照表 / C AAA / D 對立 / E 總覽）
4. **末頁一定產出鏈傑觀察 / 鏈傑洞察 / 鏈傑提醒**（依題目選詞）
5. **每頁文案從定稿提煉**（不新增論點，只精煉）

### Visual skill 必須做的事

1. **嚴守色票**：橘黃 + 黑 + 灰 + 重點藍綠
2. **嚴守字體**：思源黑體 Bold / Noto Sans CJK Bold
3. **每頁底部加 watermark**：`IG @on.chain.with.jerry`
4. **內頁標題格式統一**：粗黑體無框置中、▶ 橘色三角副標
5. **依型態選用既有版面模板**（5 種內頁型態 + 1 種封面 + 1 種末頁 + 1 種限動）
