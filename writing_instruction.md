# 鏈傑 (OnChainLens by Jerry) 寫作風格指南

這份指南是基於 Jerry 過去的高品質貼文所萃取出的寫作風格與排版慣例，適用於 IG 與 LinkedIn 的撰文。未來在協助撰寫或發想貼文時，請嚴格遵循以下原則，以保持品牌語氣的一致性與專業度。

## 1. 核心語氣與人設 (Tone & Persona)
- **專業但平易近人**：以「產業觀察者/專家」的角度出發，語氣客觀、有洞見，但不說教。
- **化繁為簡**：善用生活化的比喻來解釋艱澀的 Web3/金融科技名詞。例如：把以太坊 L1 和 L2 比喻為「央行與銀行」、把穩定幣 SCaaS 服務比喻為「白牌服務」。
- **數據與權威佐證**：經常引用大型機構（如 McKinsey, BlackRock, Visa, Coinbase）的報告、人物語錄或市場數據來增強論點的說服力。每個論點都儘量帶具體數字（如 `4.1% APY`、`30 億美元`、`T+2 交割`），避免空泛。
- **第一人稱判斷（依時機，非頻率）**：「我認為」、「我蠻看好」、「我覺得」、「在我看來」**只在提出個人觀點／給判斷／給預測時使用**，不要當作過渡句塞滿全文。全篇約 1-3 次即可。
- **代入式設問（可選）**：讓讀者從特定角色視角看問題（例如：「站在...的角度」、「對於...而言」、「想像一下...」），依題目自然度決定是否使用，切忌硬塞。

## 2. 結構模板（強烈一致）

### A. 吸引眼球的開場 (Hook)
開頭第一段一定**直接點出本文的核心張力或意義**，不浪費讀者時間。常見有三種模式：
- **事件直入 (主力)**：點名公司／事件 → 一句話總結意義。適合新產品或重大公告。
- **情境設問 (主力)**：用場景代入、丟出兩難或迷思破解。適合抽象概念或市場分析。
- **呼應前作 (極少)**：「先前寫過一篇關於...」串連系列（僅限 Jerry 明確指定時使用）。

### B. 結構化的主體 (Body)
```text
[ 開頭段：背景／設問 + 一句話定調 ]
                  │
                  ▼
[ ▌主題 1：法律/商業/技術 面向之一 ]
                  │
                  ▼
[ ▌主題 2：另一個面向 ]
                  │
                  ▼
[ ▌主題 3：（可選）第三個面向 ]
                  │
                  ▼
[ 末段：個人判斷 / 預測 / 看好結論 ]
```
- **段落符號**：大量使用 `▌` 或 `▋` 作為段落小標題的前綴，這是視覺品牌特徵。
- **三角分析框架**：寫一個議題時習慣從**多個角度**剖析，最常見的三角是：法律/監管、商業/經濟、技術/機制。
- **對比與條列**：遇到複雜概念時喜歡用「對比」說明（例如：「傳統金融機構笨重」 vs 「鏈上即時體驗」、「自建」 vs 「白牌 SCaaS」）。
- IG 貼文可適度點綴 Emoji，保持畫面乾淨專業；LinkedIn 長文則**不用** Emoji。

### C. 引發共鳴的結尾 (Call to Action / Closing)
- **LinkedIn 長文結尾**：融在末段裡的最後一句話，不另起顯式區塊。可用預測式、看好式、判斷式或意義升華式收尾。**注意：LinkedIn 文沒有寫死「鏈傑觀察」這種顯式署名。**
- **IG 貼文結尾**：必定會用一個反思或問題來引導讀者留言互動（例如：「你會在資產組合中放多少比例在比特幣呢？歡迎留言告訴我」）。末頁可用「鏈傑觀察」等顯式署名。
- **Hashtags**：置於文章最末，約 5 到 10 個精準的標籤（例如：#Web3 #區塊鏈 #金融科技 #加密貨幣）。

## 3. 論述手法與排版細節 (招牌動作)
- **具體化定錨 (Anchoring)**：將抽象龐大的數字具體化，借用大眾熟知的指標作為錨點（例如：把 4 兆美元類比為 NVIDIA 的市值），極大化 Hook 的吸引力。
- **大刀闊斧的減法**：在社群閱讀語境下，勇敢捨棄次要細節與分支挑戰。確保讀者滑動順暢，只保留最核心的比較與觀點。
- **商業化口語用詞**：使用貼近商業討論的語感（例如用「零售端接受度」取代抽象的「網路效應」），並適度加入「又」、「典型的」等字眼，增加專家點評的「人味」。
- **概念精煉**：用一個招牌詞概括複雜現象，如「甜蜜點」、「原子化」、「白牌金融」、「超 App 化」、「萬物交易所」。
- **括號吐槽（招牌特色！）**：常見半透明括號補充，讓文章有「人味」（例如：「（**很現實**）」、「（已被 Stripe 收購）」）。
- **承轉口頭禪**：「真正的痛點在於...」、「真正決定...的關鍵」、「核心門檻」、「核心價值在於」。
- **直接稱呼「你」**：代入式段落會直接說「你」（例如：「你的加密貨幣...」）。
- **專有名詞中英夾雜**：首次提到專業術語時中英文並列（如：「帳戶抽象化 (Account Abstraction, AA)」）。
- **空行留白**：段落之間必須有明顯的空行，每段 2-5 句 (約 80-180 字)，確保手機閱讀體驗舒適。

## 4. 寫作 Checklist 與絕對禁忌

### ✅ 寫作 Checklist
- [ ] 開頭點出核心張力。
- [ ] 用 `▌` 符號分 2-3 個主題段。
- [ ] 至少包含一次「三角分析」（法律/商業/技術）與「對比強化」。
- [ ] 每個論點都有具體數字或案例錨點。
- [ ] 至少打造一個「概念精煉」招牌詞。
- [ ] 至少 1-2 處「括號吐槽」式補充。
- [ ] 結尾確實帶出個人預測、判斷或看好結論。
- [ ] **定稿後，自動呼叫 `ig-cover-generator` skill 產生對應的 IG 封面圖片。**

### ❌ 典型「不像鏈傑」的反例（需注意避免）
- **避免濫用對比煽情句型**：「這不再是...而是...」。請直接陳述事實。但允許將此句型用於客觀的商業定位與價值對比（例如：「彼此不是互斥，而是找到更適合的應用場景了」）。
- **禁止自問自答**：「為什麼會這樣？因為...」、「這代表什麼？這代表...」。請直接給論點。
- **禁止使用 `——` 破折號**：請用「，」或「。」分句。
- **禁止空泛口號與過度誇張**：「這是一個革命性的突破，將徹底改變產業！」、「讓我們一起來看看這項技術。」
- **禁止躲在「業界」後面**：「對此，業界普遍認為...」。請直接寫出個人立場。
- **禁止教科書式過渡**：「總結來說...」開頭的最後段。請直接收結論。
- **禁止錯誤的第一人稱用法**：「我認為代幣化讓任何價值都能反映在鏈上」（這是事實陳述，不能用「我認為」過渡）。

## 5. 適用主題
這套寫作風格最適合以下內容領域：
- 穩定幣、央行數位貨幣 (CBDC)、代幣化資產 (RWA) 的發展趨勢
- 傳統金融 (TradFi) 與 Web3 橋接的商業邏輯與基礎設施分析
- 區塊鏈核心技術（如以太坊升級、智能合約應用）的科普與商業影響分析
- 全球金融法規與宏觀經濟事件的洞察
