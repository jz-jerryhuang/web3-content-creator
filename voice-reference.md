# 鏈傑 LinkedIn 語氣 Voice Reference v1.0

> 從 3 篇黃金樣本萃取（Coinbase 萬物交易所 / JPM Coin 登陸 Base / SCaaS 白牌金融）
> 適用：Stage 3 Writer skill 的 LinkedIn 中長文撰稿基準

---

## 一、開頭三種模式

LinkedIn 文的開頭有三種模式，**A 和 C 是主力，B 權重最低**：

| 優先 | 模式 | 特徵 | 範例 | 使用時機 |
|------|------|------|------|----------|
| ★★★ | **A. 事件直入** | 點名公司／事件 → 一句話總結意義 | 「Coinbase 是一間非常值得研究的 Web3 原生公司。在前幾天...這不只是功能的堆疊，更是一場針對傳統銀行、券商與支付清算系統的宣戰。」 | 新發表會、新產品、重大公告 |
| ★★★ | **C. 情境設問** | 用場景代入、丟出兩難或迷思破解 | 「一家傳統企業或跨國公司評估是否要導入穩定幣來服務全球客戶時，一定會面臨一個巨大難題：是要耗費龐大資源自己發行，還是直接採用...」 | 抽象概念、市場分析、無明確單一事件 |
| ★☆☆ | **B. 呼應前作** | 「先前寫過一篇關於...」串連系列 | 「先前寫過一篇關於央行數位貨幣、穩定幣、銀行存款代幣這三者的分析文章，有提到...」 | 僅限 Jerry 明確指定系列延伸，不預設使用 |

**共同點**：開頭第一段一定**直接點出本文的核心張力或意義**，不浪費讀者時間。

---

## 二、結構模板（強烈一致）

```
[ 開頭段：背景／設問 + 一句話定調 ]
                  │
                  ▼
[ ▌主題 1：法律/技術/商業面向之一 ]
                  │
                  ▼
[ ▌主題 2：另一個面向 ]
                  │
                  ▼
[ ▌主題 3：（可選）第三個面向 ]
                  │
                  ▼
[ 末段：個人判斷 / 預測 / 看好結論 ]
```

**關鍵慣例：**
- **▌符號**（黑色長方塊）做小標分節——這是視覺品牌特徵，所有 LinkedIn 文都用
- 通常 2-3 個 ▌ 主題，超過會太長
- 每個 ▌ 主題下 2-3 段，每段聚焦一個論點

## 三、論述手法（6 個招牌動作）

### 1. 具體化定錨效應 (Anchoring)
在 Hook 階段，將抽象龐大的數字與大眾熟知的實體掛鉤（例如：把 4 兆美元類比為 NVIDIA 的市值），創造具體的敬畏感與極大化 Hook 的吸引力。

### 2. 三角分析框架
寫一個議題時習慣從**多個角度**剖析。最常見的三角是：

- **法律 / 監管** — 怎麼分類？合規嗎？
- **商業 / 經濟** — 對誰有利？商業模式？
- **技術 / 機制** — 怎麼運作？技術門檻？

例：JPM Coin 文裡同時談「法律與經濟的無縫接軌」+「流動性的原子化升級」（技術機制）。

### 3. 代入式設問（**可選，非必用**）
讓讀者從特定角色視角看問題：

- 「**站在**完整掌握用戶的交易數據考量下...」（站在誰的角度）
- 「**對於** J.P. Morgan 而言...；**相對地**，**對於**持有 JPM Coin 的機構來說...」（多方視角對比）
- 「**想像一下**剛從日本客戶那邊收到美元...」（場景代入）

> **使用指引**：依題目自然度決定要不要用，**硬塞反而生硬**。Writer skill 不該每篇都強制使用。

### 4. 對比強化
喜歡把新事物 vs 舊事物擺在一起：

- 「傳統金融機構顯得既笨重又緩慢」 vs 鏈上即時體驗
- 「自建」 vs 「白牌 SCaaS」
- 「以前跨境轉帳要等 2 到 3 天，現在是秒級到帳」

### 5. 數據錨點
每個論點都儘量帶具體數字，避免空泛：

- `4.1% APY`、`30 億美元`、`T+2 交割`、`70%`、`100 億美元`
- 引第三方研究的數據（EY 9 月報告、79% 機構偏好混合模式）

### 6. 概念精煉
用一個招牌詞概括複雜現象——這是鏈傑文章的「memorable hook」：

- 「**甜蜜點**」（去中心化 vs 監管的平衡）
- 「**原子化**」（資產同步結算）
- 「**白牌**」金融（SCaaS）
- 「**超 App 化**」（Coinbase 個人金融整合）
- 「**萬物交易所**」（Everything Exchange）
- 「**日不落**財務管理」

## 四、語氣特徵

### 第一人稱判斷（**依時機，非頻率**）

「我認為」「我蠻看好」「我覺得」「在我看來」**只在提出個人觀點／給判斷／給預測時使用**——不躲在「業界認為」後面，但也別當 transition 塞滿全文。

**典型範圍：1-3 次/篇**：
- 洞察派：1-2 次（點題時、結尾預測時）
- 觀點派：2-4 次（密度可較高，但每次都要是真正的判斷）
- 科普派：1 次（多用在結尾的看好式判斷）

**對的用法**（在下判斷時）：
- 「**我認為** SCaaS 的核心價值在於...」（給結論）
- 「**我蠻看好** SCaaS 成為下一波...」（給預測）
- 「**我認為**這個切角太狹隘了。」（給立場）

**錯的用法**（不要當 transition）：
- ❌「**我認為**代幣化讓任何價值都能反映在鏈上」（這是事實陳述，不是判斷）
- ❌「**在我看來**這聽起來像技術細節」（這是描述句）

> 鏈傑風格的辨識度核心是「敢給判斷」，不是「常說我」。

### 括號吐槽 — 招牌特色！

常見半透明括號補充，讓文章有「人味」：

- 「（**很現實**）」
- 「（JPM Coin 目前尚未提供）」
- 「（已被 Stripe 收購）」
- 「（也就是薪水可以直接入帳到錢包轉成 USDC）」

> 這個括號吐槽是 Jerry 寫作的辨識度標記之一，Writer skill 應主動使用。

### 承轉口頭禪

- 「**真正的痛點在於**...」
- 「**真正決定...的關鍵**」
- 「**核心門檻**」
- 「**核心價值在於**」

### 對讀者直接稱呼「你」

代入式段落會直接說「你」：

- 「**你的**加密貨幣是由 Coinbase Inc. 託管」
- 「**你**只需要確認想下注的金額」
- 「像是**你**買一張台積電股票」

## 五、結尾收法（重要修正）

**LinkedIn 文沒有寫死「鏈傑觀察 / 鏈傑洞察 / 鏈傑提醒」這種顯式署名**——三篇樣本都沒有。LinkedIn 的個人判斷是**融在末段裡的最後一句話**，不是另起一個顯式區塊。結尾應將兩者的關係從「誰取代誰」的競爭博弈，昇華至「尋找合適應用場景 (Product-Market Fit)」的定論。

四種收法：

| 收法 | 範例 |
|------|------|
| **預測式** | 「我們最終可能會看到一個由幾家銀行巨頭主導的、雖然相對封閉但強調安全、合規的 B2B 金融生態系正在成形。」 |
| **看好式** | 「我蠻看好 SCaaS 成為下一波數位貨幣的核心基礎設施。」 |
| **適配/定位式** | 「彼此不是互斥，而是找到更適合的應用場景了。」 |
| **意義升華式** | 「CEO Jamie Dimon 展示的不僅是技術了，更像是一場銀行服務的革命。」（嵌在段中而非最末，但也是判斷句） |

> **IG carousel 末頁才用「P5-鏈傑觀察」「P8-鏈傑洞察」這類顯式署名**——這是 IG 視覺載體上的品牌標記，不等於 LinkedIn 也要那樣寫。

## 六、長度與節奏

從三篇樣本估：

- **總字數**：約 800-1100 字（介於我之前估的 500-1500 區間中段偏短）
- **段落長度**：每段 2-5 句，平均 80-180 字
- **▌主題數**：2-3 個
- **每個 ▌ 下面段落數**：2-3 段

## 七、Writer Skill 寫作 Checklist（v1.1 校正版）

撰稿時逐項檢查：

**必做**：
- [ ] 開頭是 A/B/C 三種模式之一，並在第一段定調本文核心張力
- [ ] 用 ▌ 符號分 2-3 個主題段
- [ ] 至少包含一次「三角分析」（法律/商業/技術之中至少兩面）
- [ ] 至少一次「對比強化」（傳統 vs 新、自建 vs 服務化）
- [ ] 每個論點都有具體數字／公司名／案例錨點
- [ ] 至少打造一個「概念精煉」招牌詞（題目本身就可以是一個）
- [ ] 至少 1-2 處「括號吐槽」式補充
- [ ] **第一人稱判斷 1-3 次/篇，且每次都在真正下判斷的位置**（不當 transition 用）
- [ ] **全文檢查無 `——` 破折號**（鏈傑樣本從不使用）
- [ ] 結尾用預測 / 看好 / 判斷 / 意義升華 四種之一收尾

**可選（依題目自然度）**：
- [ ] 代入式設問（站在...、對於...、想像一下...）——不必每篇都用，硬塞會生硬

**不做**：
- [ ] **不要自問自答**（「為什麼會這樣？因為...」「這代表什麼？這代表...」這種修辭禁用）
- [ ] **不要**用「鏈傑觀察」這種顯式署名（那是 IG carousel 的事，LinkedIn 不寫）
- [ ] **不用** emoji，**不用**過度形容詞，**不用**空泛口號
- [ ] **不躲**在「業界認為」「市場觀察」後面——直接寫立場

## 八、典型「不像鏈傑」的反例（避免）

| 不像鏈傑的句子 | 為什麼不對 |
|----------------|------------|
| 「這是一個革命性的突破，將徹底改變產業！」 | 過度誇張、空泛 |
| 「讓我們一起來看看這項技術。」 | 廢話開頭、沒資訊量 |
| 「Web3 是未來的趨勢，不可錯過。」 | 口號感、無具體論點 |
| 「對此，業界普遍認為...」 | 躲在「業界」後面，沒個人判斷 |
| 「總結來說...」開頭的最後段 | 太教科書式；鏈傑直接收結論不寫 transition |
| 「為什麼會發生這種事？因為...」 | **自問自答禁用**，直接給論點 |
| 「這代表什麼？這代表 X 即將...」 | **自問自答禁用**，改寫成「真正的關鍵在於 X」 |
| 「會帶來什麼影響？答案是...」 | **自問自答禁用**，直接陳述影響 |
| 任何含 `——` 雙連字符的句子 | **`——` 全面禁用**，用「，」「。」分句 |
| 「**我認為**代幣化讓任何價值都能反映在鏈上」 | 第一人稱**不能當 transition**——這是事實陳述不是判斷 |
