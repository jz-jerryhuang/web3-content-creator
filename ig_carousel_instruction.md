# 鏈傑 IG Carousel 分鏡腳本 (Storyboard) 指南

當 IG 或 LinkedIn 的長文/初稿確認後，下一步是將文字轉化為適合 Instagram 閱讀的「多圖輪播 (Carousel) 分鏡腳本」。請根據以下指南，將長文拆解並設計為 3 到 10 頁的圖像結構。

## 1. 總體結構與文字轉換原則
- **取消過渡句**：捨棄長文中的起承轉合（如「真正的痛點在於...」），直接利用視覺排版與翻頁來分隔論點。
- **極度濃縮**：每頁的文案必須極度精煉，不要大段落文字。全篇字數應控制在 600-1000 字以內。
- **符號差異**：
  - **不使用** LinkedIn 常用的 `▌` 或 `▋` 符號。
  - 大量使用 **▶ 橘色三角** 作為副標題前綴。
  - 將「概念精煉招牌詞」（如：甜蜜點、BaaS 2.0、適者生存）獨立成為畫面上的標籤或引號金句卡。

## 2. 封面 (P1) 公式
封面必須具有極強的張力，不能是空泛口號。

### 標題公式
- **爭議題**：「主標：副標問句？」 (例：穩定幣戰場解析：是革命還是威脅？)
- **解釋題**：「從 X 到 Y：定論短句」 (例：從交易所到基礎設施：Coinbase 萬物交易所)
- **預測題**：「未來 X 年的 Y？...」 (例：未來十年的 Alpha？Google 新支付協議 AP2)
- **抒情題**：「主標，副標問句」 (例：危險邊緣的美元，穩定幣能成為救命稻草嗎？)

### 視覺建議 (Canva Design Note)
- **A. 個人實拍 + 浮動元素**（適合較柔軟、具作者人格的主題）。
- **B. AI 生成概念場景**（適合宏觀、重型議題，如宇宙球體、棋盤）。

> **💡 自動化產圖提示**：當 IG 貼文/腳本「定稿」後，請自動呼叫 `ig-cover-generator` skill，將文章標題與內容輸入，直接生成符合此視覺建議的 3D/數位藝術風格封面圖。

## 3. 內頁公式 (P2 - Pn-1)
將長文內容依據邏輯套入以下 5 種「內頁型態模板」中：

- **型態 A：多卡片網格 (Grid)**
  - **適用**：列舉多個案例、多國比較、動態統整。
  - **結構**：大標 + ▶ 子標題 + 2x2 卡片（各含 30-50 字）+ 底部圖表數據。
- **型態 B：對照表 (Table) 🏆 最強招牌**
  - **適用**：概念比較、多方案對比（複雜概念一眼看懂）。
  - **結構**：大標 + 淺灰副標 + 多欄對照表（格內短文字+顏色標記）+ 底部一句結論。
- **型態 C：精準論點拆解 (A.A.A. 結構)**
  - **適用**：技術/概念深度拆解。
  - **結構**：大標 + 場景白話引述 + 三點大寫英文字母論述（如 Authorization, Authenticity）+ 解決方案數字 1/2/3。
- **型態 D：對立敘事 (Opposing Views)**
  - **適用**：辯證題、雙面影響分析。
  - **結構**：問句標題 + 左/右兩邊對立圓角按鈕（綠/橘色區分）+ 中央視覺切割線 + 底部粗體金句總結。
- **型態 E：總攬 (Overview)**
  - **適用**：倒數第二頁的全局統整。
  - **結構**：大標 + 大型對照表 + 底部趨勢標籤與行動建議。

## 4. 末頁收尾公式 (Pn)
Carousel 的最後一頁（不一定要死守「鏈傑觀察」，請依題目自然度選擇）：

- **型態 A：鏈傑觀察 / 洞察 / 提醒 (專屬署名頁)**
  - **結構**：以「鏈傑觀察」為大標，使用 ▶ 分列 2-3 個觀察點。
  - **文字**：這裡是完整論述段落（約 200-300 字），不是條列點。結尾必須是個人判斷或預測，可插入引號金句卡。
- **型態 B：戰場總攬 / 比較表**
  - **適用**：競爭題、總覽題，用對照多方勢力作結更有力。
- **型態 C：趨勢標籤 + 行動建議**
  - **適用**：給讀者明確行動方向時（如「擁抱合規／深耕全球南方」）。

## 5. 視覺品牌系統 (給視覺設計師/Canva 的提示)
- **比例**：Carousel 4:5 直式。
- **字體**：粗黑體 (思源黑體 Bold / Noto Sans CJK)。
- **色票**：橘黃 (主標/強調)、黑 (內文標題)、灰 (內文補述)、綠/紅橘 (對立立場)、藍/青 (次要強調)。
- **浮水印**：每頁底部置中必須有 `IG @on.chain.with.jerry`。

## 6. Canva 大量建立 (Bulk Create) 專用 CSV 格式
為了實現半自動化產圖，Agent (`copywriter`) 輸出最終腳本時，必須提供標準的 CSV 格式。你可將此 CSV 複製存檔後，直接匯入 Canva 的「大量建立」功能。

**CSV 欄位定義**：
1. `Page`: 頁碼 (P1, P2...)
2. `Template_Type`: 內頁型態 (Cover, Grid, Table, AAA, Opposing, Overview, Observation)
3. `Main_Title`: 該頁的大標題
4. `Sub_Title`: 該頁的副標題 (▶ 後面的文字)
5. `Content`: 內文 (以 `\n` 或 Canva 的 Alt+Enter 換行，多行文字須用雙引號包覆)
6. `Footer_Text`: 底部強調金句
7. `Design_Note`: 給 Canva 設計的提詞或元素搜尋建議

**範例輸出**：
```csv
Page,Template_Type,Main_Title,Sub_Title,Content,Footer_Text,Design_Note
P1,Cover,危險邊緣的美元,穩定幣能成為救命稻草嗎？,,,AI Generated waterfall background
P2,Grid,多重挑戰,美元霸權面臨的三個危機,"- 國家1...\n- 國家2...",,Checkmark icons
```
